# ろるさぽくん — Riot Games規約準拠・配信可能性調査レポート

## 調査の背景
「ろるさぽくん」をアプリケーションとして配信・収益化してよいか、Riot Gamesの利用規約・開発者ポリシーの観点から調査する。

---

## 1. 総合評価

**結論: 基本的に配信・収益化は可能。ただし、いくつかの対応事項あり。**

「ろるさぽくん」はRiot Gamesの開発者ポリシーの趣旨に概ね合致している。Riotが推奨する「プレイヤーの成長を支援するツール」そのものである。ただし、以下の対応が必要。

---

## 2. 準拠している点（Green）

| 項目 | 状況 | 根拠 |
|------|------|------|
| **使用API** | Live Client Data API + LCU API（いずれもローカル、認証不要/Basic認証）| Riot公式APIキーを使用していないため、APIキー関連の制約は直接適用されない |
| **ゲーム操作の自動化なし** | アドバイス表示のみ、プレイヤーの操作に介入しない | ボット・スクリプト・チート・MODに該当しない |
| **プレイヤーの意思決定を残している** | 候補提示＋理由説明であり、最終判断はプレイヤーが行う | "removal of player decision-making" に該当しない |
| **無料ティアの提供** | Ollama（ローカルLLM）で完全無料利用可能 | 収益化の前提条件「無料アクセス」を満たす |
| **Data Dragon使用** | 公式CDNから静的データ取得 | 公開データであり制限なし |
| **プレイヤーデータの脱匿名化なし** | サモナー名以外の個人情報を取得・表示しない | プライバシーポリシー準拠 |

---

## 3. 要対応事項（Yellow — 対応すれば問題なし）

### 3-1. Developer Portal への登録（必須）
- **現状**: 未登録（設計書に「配布時はRiot Developer Portalへのアプリ登録を予定」と記載）
- **必要なアクション**: https://developer.riotgames.com でアプリケーションを登録し、審査を通す
- **備考**: Live Client Data APIとLCU APIはローカルAPIなのでAPIキーは不要だが、Riot IPを使用するアプリケーションとしての登録は必要

### 3-2. 免責事項の表示（必須）
- **現状**: 対応済み（SettingsDialog + LegalDialog に掲載）
- **掲載文言**:
  > 「ろるさぽくん isn't endorsed by Riot Games and doesn't reflect the views or opinions of Riot Games or anyone officially involved in producing or managing Riot Games properties. Riot Games, and all associated properties are trademarks or registered trademarks of Riot Games, Inc.」
- **掲載場所**: アプリ内（設定画面 + 免責事項ページ）+ 配布ページ（要対応）

### 3-3. 収益化モデルの適正化
- **現状**: フリーミアム（Free + Pro月額）を計画
- **Riot要件**:
  - 無料ティア必須 → **満たしている**（Ollamaローカルモード）
  - 「transformative value」の提供 → **満たしている**（AIによる分析・洞察は新しい価値）
  - 「gouging（過度な価格設定）」禁止 → Pro価格は妥当な範囲に設定する必要あり
- **注意**: 有料版と無料版の機能差が大きすぎると「ペイウォール」とみなされるリスクがある。現設計ではFree版でも全5機能が使えるため問題ない

### 3-4. APIキーのハードコード禁止
- **現状**: AWS Bedrock認証情報がビルド時にasarに埋め込まれる設計
- **リスク**: Riotポリシーは「APIキーを配布コード内にハードコード禁止」としているが、これはRiot APIキーに関する規定。AWS/Anthropicのキーについてはセキュリティ上の問題であってRiot規約違反ではない
- **推奨**: とはいえ、API Gateway経由に移行する既存計画を推進すべき

## 4. リスク分析（Orange — グレーゾーン）

### 4-1. 「ゲームを解く（solve）」ツールとみなされるか？

**Riotの方針**:
> "We want to see products that help players get better at our games, or track their growth. What we don't want to see are products that solve our games or make everything too simple."

**分析**:
- **「ろるさぽくん」は「コーチ」であり「自動操縦」ではない**
  - アイテムは候補＋理由の提示であり、自動購入しない
  - マクロアドバイスは提案であり、プレイヤーが判断する
  - 試合後コーチングは振り返りであり、リアルタイム操作に影響しない
- **類似ツールとの比較**:
  - Blitz.gg、Porofessor、Mobalytics等は試合中のリアルタイムオーバーレイを提供しており、Riotに認められている
  - これらのツールもアイテムビルド推奨、対面情報表示を行っている
- **AIによるアドバイスの程度**:
  - 静的な統計データ表示（Blitz等と同等）に加え、試合展開に応じた動的アドバイスを提供
  - これは「より高度なコーチング」であり、「ゲームを解く」とまでは言えない
  - ただし、あまりに具体的・即時的な指示（「今すぐここに移動しろ」等）は避けるべき

**結論**: 現在の設計は許容範囲内と判断。ただし、アドバイスの粒度が「提案」を超えて「指示」にならないよう注意が必要。

### 4-2. LLM/AIの利用に関する明示的な規定がない

- Riot Gamesの開発者ポリシーにはAI/LLMの利用に関する**明示的な規定がない**
- これは「許可」でも「禁止」でもなく、**未規定**の状態
- AIを使ったLoLツールは比較的新しい分野であり、今後ポリシーが更新される可能性がある
- **推奨**: Developer Portal登録時にAI機能について明示的に説明し、承認を得る

### 4-3. OP.GG MCP APIの利用

- OP.GGはMIT Licenseで公開しているMCP Server
- OP.GG自体がRiotから認められたサービスであり、その公開APIを使うこと自体は問題ない
- ただし、OP.GGの利用規約も別途確認すべき

## 5. 問題なし（明確にセーフ）

| 項目 | 理由 |
|------|------|
| ギャンブル・ベッティング要素 | なし |
| クリプトカレンシー・ブロックチェーン | なし |
| Riot IPを使ったゲーム開発 | ゲームではなくツール |
| 逆エンジニアリング | Live Client Data APIは公式公開API |
| 不正な優位性の作成 | 公開情報の分析のみ、隠しデータへのアクセスなし |
| 広告表示 | Riotポリシーで許可されている収益化方法 |

## 6. 道徳的・倫理的観点

### ポジティブな側面
- **初中級者の学習支援**: ゲーム理解を深め、プレイヤーの成長を促進
- **情報格差の解消**: 上級者が当たり前に知っている知識を初心者にも提供
- **無料アクセス可能**: 経済的に余裕のないプレイヤーも利用できる

### 注意すべき側面
- **スキル発達への影響**: 常にAIに頼ると自力で判断する力が育たない可能性 → コーチング機能で「なぜ」を説明する設計で緩和
- **対戦相手との公平性**: 使用者が有利になる → ただし同種のツール（Blitz、Porofessor）は既に広く普及しており、業界標準

## 8. 結論

**「ろるさぽくん」はRiot Gamesの開発者ポリシーに基本的に準拠しており、配信・収益化は可能と判断。**

最大のリスクは「ゲームを解くツール」とみなされることだが、現在の設計（提案型、理由付き、プレイヤー判断尊重）であれば、Blitz.gg等の既存認定ツールと同じカテゴリに収まる。AI/LLM利用に関する明示的な規定はないため、Developer Portal登録時にRiotに直接確認することを強く推奨する。
