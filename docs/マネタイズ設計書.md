# マネタイズ設計書

## 1. 概要

常にクラウドAI（Gemini）を使用する構造。課金はMicrosoft Storeのサブスクリプション機能で管理し、APIキーの保護にLambda Function URLを1つ使用する。

- 無料プラン: 1日5回までAI利用可能
- Proプラン: AI利用無制限（¥980/月）
- ログイン・ライセンスキー不要（Storeが管理）

### Early Access 期間

初期リリース（Early Access）では特典としてAI利用回数を無制限とする。次期アップデートで無料/有料の回数制限を導入する。

## 3. 全体アーキテクチャ

```
アプリ                   Microsoft Store          Lambda Function URL        Gemini API
  │                         │                         │                        │
  │ 「Proにする」            │                         │                        │
  │ ──────────────────────►│                         │                        │
  │ ◄── Store購入ダイアログ ─│                         │                        │
  │ カード入力・購入完了      │                         │                        │
  │ ◄── サブスク有効 ────── │                         │                        │
  │                         │                         │                        │
  │ AI機能利用時             │                         │                        │
  │ ─── POST /generate ──────────────────────────── ►│                        │
  │                         │                         │ Gemini APIキーを付与     │
  │                         │                         │ ─────────────────────► │
  │                         │                         │ ◄── AI応答 ──────────  │
  │ ◄── AI応答 ─────────────────────────────────────  │                        │
```

### ポイント

- **課金**: Microsoft Storeが全管理（決済・解約・更新）
- **APIキー保護**: Lambda Function URLがプロキシとなり、Gemini APIキーをクライアントに露出させない
- **プラン判定**: アプリ側でStore APIを使いローカルにライセンス状態を確認。日次カウンターもローカル管理

## 5. AWSインフラ構成

### アーキテクチャ

```
Lambda Function URL (gemini-proxy)
  └─ POST /generate  → Gemini APIへ中継

Secrets Manager
  └─ gemini-api-key   (Gemini APIキー)
```

### Lambda関数: gemini-proxy

- トリガー: アプリからPOST（Lambda Function URL）
- 処理:
  1. リクエストを受信
  2. Secrets ManagerからGemini APIキーを取得（キャッシュ）
  3. Gemini APIにリクエストを中継
  4. レスポンスをアプリに返却
- 簡易的な保護:
  - アプリ固有のシークレットをヘッダーに付与して検証
  - IPあたりのレート制限（Lambda内で簡易実装）

### 月額コスト見込み

| サービス | 月額 | 備考 |
|---------|------|------|
| Lambda | $0 | 無料枠: 月100万リクエスト |
| Secrets Manager | $0.40 | 1シークレット × $0.40 |
| **合計** | **≈ $0.40/月** | ユーザー数千人規模まで |

## 6. アプリ側の変更点

### preload.js（追加IPC）

- `store:checkLicense` → Storeのサブスクリプション状態を確認
- `store:purchase` → Store購入ダイアログを表示
- `ai:generate` → Lambda Function URL経由でGemini APIを呼び出し

### SettingsDialog.jsx

- 現在のプラン表示（Free / Pro）
- 無料プランの場合「Proにアップグレード」ボタン → Store購入ダイアログ
- 本日の残り回数を表示（Freeプランの場合）

### main.js

- 起動時にStore APIでライセンス状態を確認
- AI利用時に日次カウンターをチェック（ローカル管理、日付変更でリセット）
- 無料プラン: 1日5回超過で利用制限 → アップグレード誘導
- Proプラン: 回数制限なし

## 7. 実装優先順

| Phase | 内容 | 依存 |
|-------|------|------|
| M1 | CDKスタック + gemini-proxy Lambda | Gemini APIキー取得 |
| M2 | アプリ側AI呼び出しをLambda経由に変更 | M1 |
| M3 | Microsoft Storeサブスクリプション設定 + アプリ側Store API連携 | Storeアカウント |
| M4 | Microsoft Store提出 | M2, M3 |

## 8. 不正利用対策

| リスク | 対策 |
|--------|------|
| Lambda URL直叩き | アプリ固有シークレットをヘッダーで検証 + IPレート制限 |
| 日次回数リセット不正 | 日次カウンターはローカル管理（日付ベース）。厳密な不正対策はユーザー規模に応じて検討 |
| サブスク共有 | Microsoft Storeのアカウント単位で管理されるため、Store側が制御 |

初期段階では過剰対策不要。ユーザー規模に応じて段階的に強化する。
